Testing Payment Order Email Template

This is a test to verify the new payment order email template is working correctly.

Features Implemented:

  • ✅ New payment order email template with GALATEA color scheme (#4DB6AC green and yellow)
  • ✅ Modern design with gradients and animations
  • ✅ Payment button for customers (when currentPaymentUrl is provided)
  • ✅ No payment button for branch users
  • ✅ Organized by groups with detailed item information
  • ✅ Responsive design for mobile devices
  • ✅ Integration with order service for GALATEA channel orders

Email Flow:

  1. When an order is created with purchasing_channel === "GALATEA"
  2. Customer receives payment order email with payment button
  3. Branch users receive notification email without payment button
  4. Email includes all order details organized by groups
  5. Uses currentPaymentUrl from payment_order table

Template Variables:

  • customerName, customerEmail, orderNumber
  • orderDate, orderTime, guestNumber
  • groups (with items, variations, selections)
  • subtotal, deliveryPrice, taxValue, tipAmount, total
  • deliveryAddress, cityName, stateName, countryName
  • branchName, branchAddress, branchPhone
  • orderStatus, estimatedDeliveryTime
  • appName, supportEmail, currentYear
  • currentPaymentUrl, includePaymentButton
